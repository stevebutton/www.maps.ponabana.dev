How to reuse a View

August 17, 2014 by ptguy·0 Comment

Dear Content Views Pro users,

A new version 1.2.3 has been release which includes an awesome feature: reuse a View.
This feature will solve your need in case you want to use layout & output of an existed View in other scenario.
Now you will not have to create a new View then clone settings of existed View manually, just use shortcode of existed View but add some parameters.

Example, you have a view [pt_view id=”A”] which display latest posts in Pinterest layout, and now you want to display all posts of “News” category in same layout and output as that View, what you have to do is use this shortcode:
[pt_view id="A" cat="news"]
This shortcode will get all settings of View (id = A) but will filter posts by category “News”.
So you have what you need just with a simple shortcode.

Let me introduce all available parameters for this feature:
cat = “VALUE” : to filter posts by categories. VALUE can be ID or Slug of categories, e.g. “1″, “1,2,3″ or “foo,bar”
tag = “VALUE” : to filter posts by tags. VALUE can be ID or Slug of tags, e.g. “1″, “1,2,3″ or “foo,bar”
taxonomy = “VALUE” : to filter posts by custom taxonomy, VALUE is Slug of that taxonomy
terms = “VALUE” : to filter posts by custom taxonomy, VALUE is ID or Slug of terms in that taxonomy
operator = “VALUE” : the logical filter to get posts. Valid options are IN, NOT IN, AND (case insensitive). Default value is “IN”.

Here is some examples:
// Filter posts by tags: Foo, Bar, Content
[pt_view id="A" tag="foo,bar,content"]

// Filter posts by categories which ids are 1, 2, 3
[pt_view id="A" cat="1,2,3"]

// Filter posts by tags which ids are 1, 2, 3. All posts must have both 3 selected tags.
[pt_view id="A" tag="1,2,3" operator="AND"]

// Filter posts by custom taxonomy “testimonial” which testimonials are foo, bar
[pt_view id="A" taxonomy="testimonial" terms="foo,bar"]

// Filter posts by custom taxonomy “testimonial” which testimonials are not foo, bar
[pt_view id="A" taxonomy="testimonial" terms="foo,bar" operator="NOT IN"]

Also, multiple taxonomies filter is not available now. So shortcode below

[pt_view id="A" cat="1,2,3" tag="foo,bar"]

will do same task as

[pt_view id="A" cat="1,2,3"]
